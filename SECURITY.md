# Security Policy

Thank you for helping keep Yumil MPM users safe.

Yumil MPM のユーザーを守るためのご協力に感謝します。

---

## Reporting a vulnerability / 脆弱性の報告

Please do **not** open a public GitHub Issue for security-sensitive matters.

セキュリティに関わる事項は **公開 Issue を使わず**ご報告ください。

Instead, please email:

代わりに、以下までメールでご連絡ください:

> **yumil.mpm@gmail.com**

When reporting, the following information is helpful:

報告時には以下をお書きいただけると助かります:

- A description of the issue / 問題の概要
- Steps to reproduce (if known) / 再現手順（分かれば）
- Affected version(s) / 影響するバージョン
- Any suggested fix or workaround / 修正案・回避策（あれば）

---

## What to expect / 対応の流れ

- Initial reply may take some time; thank you for your patience.
- 初動対応にはお時間をいただく場合があります。ご了承ください。
- Once confirmed, a fix will be prepared and shipped through the normal release channels (Microsoft Store / GitHub Releases).
- 確認できた問題は通常のリリースチャネル（Microsoft Store / GitHub Releases）で修正配信します。
- After the fix is released, the issue may be disclosed publicly, with credit to the reporter where appropriate (and with their consent).
- 修正リリース後、報告者の同意のもとで公開化し、ご希望に応じてクレジットを記載することがあります。

---

## Scope / 対象範囲

This policy covers **Yumil MPM itself** (the application distributed via Microsoft Store and GitHub Releases).

このポリシーは **Yumil MPM 本体**（Microsoft Store および GitHub Releases で配布されるアプリケーション）を対象とします。

For related items, the appropriate contact differs:

関連する以下のものについては、お問い合わせ先が異なります:

- **Third-party dependencies** (libraries used by Yumil MPM): please report to the relevant upstream project.
- **サードパーティ依存ライブラリ**（Yumil MPM が利用するライブラリ）: 該当する上流プロジェクトへのご報告をお願いします。
- **Individual custom plugins** distributed by third-party developers: please contact the developer or distributor of each plugin.
- **第三者の開発者が配布する個別のカスタムプラグイン**: 各プラグインの開発元・配布元にお問い合わせください。

If you are unsure whether something falls under this policy, feel free to email anyway and we will help route it.

このポリシーの対象範囲かどうか判断がつかない場合も、まずはメールでお気軽にご連絡ください。適切な連絡先をご案内します。
